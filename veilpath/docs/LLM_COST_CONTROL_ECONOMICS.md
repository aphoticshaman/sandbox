# LLM Cost Control Through Game Economics
## How to Make AI Oracle Readings Profitable

**Core Insight:** If we gate tarot readings and LLM-oracle chats behind currency, quests, reputation levels, and daily limits, we'll never have to worry about tokens being used on our end versus our overhead versus our revenue/profits.

---

## The Problem: Unlimited LLM Access Kills Profits

**Traditional AI App Model (BROKEN):**
```
User pays $4.99/month
→ Uses LLM 100+ times/day
→ Token costs: $50-100/month
→ We LOSE money on power users
```

**Why this fails:**
- Power users abuse unlimited access
- Token costs scale linearly with usage
- No way to control spending
- Profit margins disappear
- Forced to raise prices or restrict heavily

---

## The Solution: Game Economy Gates

**VeilPath Tarot Model (PROFITABLE):**
```
User pays $4.99/month subscription
→ Earns currency through gameplay (time investment)
→ Spends currency on readings (limited by earning rate)
→ Daily caps prevent abuse
→ Token costs: $2-5/month (controlled)
→ We PROFIT $2.99 - $0.01 per user = 40-60% margin
```

**Why this works:**
- Users FEEL like they get unlimited access (earn more currency)
- But actual usage is gated by time/gameplay
- Token costs are predictable and capped
- Engagement increases (playing to earn)
- Perceived value is HIGH (worked for it)

---

## Gating Mechanisms

### 1. Daily Limits (Hard Cap)

**Free Users:**
- 2 readings per day (resets midnight)
- Max 3-card spreads
- Basic LLM interpretation (shorter, cached responses)
- Token cost per user: ~$0.10/day = $3/month
- Revenue: $0 (free tier)
- **Result: Loss leader, converts to paid**

**Subscribers:**
- 10 readings per day (resets midnight)
- All spread types unlocked
- Deep LLM interpretation (full context, personalized)
- Token cost per user: ~$0.20/day = $6/month
- Revenue: $4.99/month
- **Result: LOSS on heavy users (need more gates)**

**Solution: Add secondary gates below**

### 2. Currency Cost (Soft Cap)

**Reading Prices in Veil Shards:**

```
Single Card Reading: 10 shards (cheapest)
├─ Token cost: ~500 tokens ($0.01)
├─ Interpretation: 1 paragraph
└─ Time: 2 minutes

Three Card Spread: 25 shards
├─ Token cost: ~1500 tokens ($0.03)
├─ Interpretation: 3 paragraphs
└─ Time: 5 minutes

Celtic Cross: 50 shards
├─ Token cost: ~3000 tokens ($0.06)
├─ Interpretation: 10 cards, deep analysis
└─ Time: 15 minutes

Year Wheel: 100 shards
├─ Token cost: ~5000 tokens ($0.10)
├─ Interpretation: 12 cards, extensive
└─ Time: 30 minutes

Soul Reading (ultimate): 200 shards
├─ Token cost: ~10000 tokens ($0.20)
├─ Interpretation: Full life analysis
└─ Time: 60 minutes
```

**Daily Earning Cap:**
```
Subscribers can earn: 100 shards/day maximum
├─ Login: 10 shards
├─ Quests: 40 shards (time investment)
├─ NPCs: 20 shards (relationship building)
├─ Achievements: 30 shards (one-time bonuses)
└─ Total: 100 shards/day

This allows:
- 10 single cards OR
- 4 three-card spreads OR
- 2 Celtic Cross OR
- 1 Year Wheel

Token cost if maxed: $0.10 - 0.60/day depending on choices
```

**Result: Users can do LOTS of readings, but must EARN the currency**


**Result: Highest token costs are for most engaged users who've invested weeks/months**

### 4. Quest-Based Unlocks (Time Investment)

**Premium Readings Require Quests:**

```
"Unlock the Fae Court Reading" Quest:
├─ Complete 5 NPC quests
├─ Earn 80 reputation with Luna
├─ Pay 150 shards
└─ Time investment: 3-5 hours gameplay

Reward:
├─ Fae Court Spread (special 7-card layout)
├─ Token cost: ~4000 tokens ($0.08)
└─ Player feels they EARNED this special reading
```

**Result: Players spend TIME instead of causing TOKEN COSTS repeatedly**

### 5. Cooldown Timers (Engagement Pacing)

**Deep Readings Have Cooldowns:**

```
Celtic Cross: 24-hour cooldown
├─ Can only do once per day
├─ Even if you have currency
└─ Prevents spam, ensures reflection

Year Wheel: 7-day cooldown
├─ Once per week maximum
├─ Encourages planning when to use
└─ High token cost spread becomes special event

Soul Reading: 30-day cooldown
├─ Once per month

```

**Result: Most expensive readings naturally limited by time**

## Additional Cost Controls

### 6. Cached Responses (Template System)

**Not every reading needs fresh LLM generation:**

```javascript
// Check if we have similar reading cached
const cacheKey = `${card.id}_${isReversed}_${intention_type}_${user_rep_tier}`;

if (cache.has(cacheKey) && user.reputation < 60) {
  // Casual users get cached interpretations (no token cost)
  return cache.get(cacheKey);
} else {
  // High-rep users get fresh, personalized LLM generation
  return await generateLLMReading(fullContext);
}
```

**Savings:**
- 50-70% of readings can use cached templates
- Only deep, high-rep readings need fresh generation
- Token costs drop significantly

### 7. Tiered LLM Models

**Not every reading needs GPT-4:**

```javascript
const modelSelection = {
  singleCard_lowRep: 'gpt-3.5-turbo', // $0.002 per 1k tokens
  threeCard_midRep: 'gpt-3.5-turbo',
  celticCross_highRep: 'gpt-4-turbo', // $0.01 per 1k tokens
  yearWheel_maxRep: 'gpt-4-turbo',
  soulReading_beloved: 'gpt-4-turbo',
};
```

**Savings:**
- 70% of readings use cheaper model
- Premium experiences justify premium model
- Average token cost drops 40-60%

### 8. Length Limits by Type

**Control output verbosity:**

```javascript
const responseLimits = {
  singleCard: { maxTokens: 150 },      // 1 paragraph
  threeCard: { maxTokens: 400 },       // 3 paragraphs
  celticCross: { maxTokens: 1000 },    // Deep but controlled
  yearWheel: { maxTokens: 1500 },      // Extensive
  soulReading: { maxTokens: 2500 },    // Ultimate
};
```

**Savings:**
- Prevents runaway token generation
- Users still get value
- Costs predictable

## Risk Mitigation

**What if users abuse the system?**

**Exploit 1: Bot farming shards**
- Solution: Require real gameplay (not just API calls)
- Captcha for high-frequency actions
- Ban suspicious accounts

**Exploit 2: Multiple accounts**
- Solution: Device fingerprinting
- Email verification
- Limit free tier value

**Exploit 3: Sharing accounts**
- Solution: Session limits (1 device at a time)
- Flag simultaneous logins
- Encourage individual subscriptions (profiles, progress)

**If costs still too high:**
- Reduce daily shard cap (100 → 75)
- Increase reading prices (10 → 15 shards)
- Shorten LLM response lengths
- Use cheaper models more aggressively
- Add more cooldowns

**We have MANY levers to pull.**
